# i18n (frontend UI shell) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract every frontend UI string into a typed, namespaced i18n layer and ship 7 languages (en, zh-Hans, ja, es, de, fr, ko), with pt-BR/ar registered but not yet selectable.

**Architecture:** react-i18next with statically-bundled JSON resources per (locale, namespace). A pure locale *registry* (`config.ts`) is the single source for supported locales, their direction, and nearest-match resolution. Current locale lives in the existing `useUiStore` (mirroring `theme`); a `LocaleProvider` effect syncs it to i18next + `<html lang>`/`<html dir>`. English JSON is the authoritative key set; a parity test forces every other locale to match it; an ESLint rule forbids new hardcoded JSX strings.

**Tech Stack:** TypeScript, React 18, Vite, Tailwind, Zustand, `i18next`, `react-i18next`, `eslint-plugin-i18next` (dev). No language-detector dependency — detection is a few lines we own.

## Global Constraints

- Design principles: **simple, explicit, clear, complete** — no unnecessary entities.
- All project files and code are **English** (pure-English project); the *only* non-English text lives in `locales/<code>/*.json` values.
- Package name stays `@ai4s/desktop`; bundle id / `@ai4s/*` unchanged. Display brand is "Open Science".
- The UI never calls OpenCode directly (unaffected here — no runtime calls added).
- Scope is the **frontend UI shell only**. Do NOT translate: AI-generated content, Rust-side strings, or `examples/` prose.
- Work happens on branch `feat/i18n` (already created).
- All paths below are relative to `apps/desktop/` unless stated. Alias `@` → `apps/desktop/src`.
- Run commands from `apps/desktop/`. Test runner: `pnpm test` (vitest). Typecheck: `pnpm typecheck`. Lint: `pnpm lint`.
- Namespaces (fixed set): `common`, `nav`, `settings`, `runs`, `session`, `inspector`, `errors`.
- Locale codes & display order: `en`, `zh-Hans`, `ja`, `es`, `de`, `fr`, `ko` (shipped) then `pt-BR`, `ar` (registered, `shipped: false`; `ar` is `dir: rtl`).

Create `src/i18n/config.ts`:

```ts
export type Direction = "ltr" | "rtl";

export interface LocaleMeta {
  /** BCP-47 tag; also the i18next language key. */
  code: string;
  /** English name — for developer reference, never shown to users. */
  label: string;
  /** Endonym shown in the language switcher. */
  nativeName: string;
  dir: Direction;
  /** false = registered (direction/keys known) but not yet selectable. */
  shipped: boolean;
}

export const DEFAULT_LOCALE = "en";

/** localStorage key holding the user's chosen locale. */
export const LOCALE_KEY = "ai4s.locale";

/** Registration order is the switcher's display order. */
export const LOCALES: LocaleMeta[] = [
  { code: "en", label: "English", nativeName: "English", dir: "ltr", shipped: true },
  { code: "zh-Hans", label: "Simplified Chinese", nativeName: "简体中文", dir: "ltr", shipped: true },
  { code: "ja", label: "Japanese", nativeName: "日本語", dir: "ltr", shipped: true },
  { code: "es", label: "Spanish", nativeName: "Español", dir: "ltr", shipped: true },
  { code: "de", label: "German", nativeName: "Deutsch", dir: "ltr", shipped: true },
  { code: "fr", label: "French", nativeName: "Français", dir: "ltr", shipped: true },
  { code: "ko", label: "Korean", nativeName: "한국어", dir: "ltr", shipped: true },
  { code: "pt-BR", label: "Portuguese (Brazil)", nativeName: "Português (Brasil)", dir: "ltr", shipped: false },
  { code: "ar", label: "Arabic", nativeName: "العربية", dir: "rtl", shipped: false },
];

export function shippedLocales(): LocaleMeta[] {
  return LOCALES.filter((l) => l.shipped);
}

export function localeMeta(code: string): LocaleMeta | undefined {
  return LOCALES.find((l) => l.code === code);
}

/** Map an arbitrary BCP-47 tag to the nearest SHIPPED locale:
 *  exact (case-insensitive) → same base language → DEFAULT_LOCALE.
 *  Unshipped locales are never a resolution target. */
export function resolveLocale(candidate: string | null | undefined): string {
  if (!candidate) return DEFAULT_LOCALE;
  const shipped = shippedLocales();
  const want = candidate.toLowerCase();
  const exact = shipped.find((l) => l.code.toLowerCase() === want);
  if (exact) return exact.code;
  const base = want.split("-")[0];
  const byBase = shipped.find((l) => l.code.toLowerCase().split("-")[0] === base);
  return byBase ? byBase.code : DEFAULT_LOCALE;
}

/** First-run guess when no preference is stored: saved value → OS/browser
 *  language → default. Persistence is owned by the store; this only seeds it. */
export function detectInitialLocale(): string {
  if (typeof window === "undefined") return DEFAULT_LOCALE;
  const saved = window.localStorage.getItem(LOCALE_KEY);
  if (saved) return resolveLocale(saved);
  const nav = typeof navigator !== "undefined" ? navigator.language : null;
  return resolveLocale(nav);
}
```

Create `src/app/providers/LocaleProvider.tsx`:

```tsx
import { useEffect, type ReactNode } from "react";
import { useUiStore } from "@/lib/store";
import { localeMeta } from "@/i18n/config";
import i18n from "@/i18n";

/** Syncs the store's locale into i18next and onto the document root.
 *  Direction is set once here from the locale registry — components must never
 *  hardcode direction (see the i18n design doc, §7 RTL readiness). */
export function LocaleProvider({ children }: { children: ReactNode }) {
  const locale = useUiStore((s) => s.locale);
  useEffect(() => {
    void i18n.changeLanguage(locale);
    document.documentElement.lang = locale;
    document.documentElement.dir = localeMeta(locale)?.dir ?? "ltr";
  }, [locale]);
  return <>{children}</>;
}
```

## Task 11: Key-parity test (safety net)

Force every non-English locale to carry exactly English's key set. This fails until translations are added (Task 12), so it is written first as the gate.

**Files:**
- Create: `src/i18n/parity.test.ts`

**Interfaces:**
- Consumes: `i18n` (`getResourceBundle`), `NAMESPACES`, `shippedLocales`.

- [ ] **Step 1: Write the test**

Create `src/i18n/parity.test.ts`:

```ts
import { describe, expect, it } from "vitest";
import i18n, { NAMESPACES } from "./index";
import { DEFAULT_LOCALE, shippedLocales } from "./config";

/** All dot-paths of leaf string keys in a nested object. */
function keyPaths(obj: unknown, prefix = ""): string[] {
  if (obj === null || typeof obj !== "object") return [prefix];
  return Object.entries(obj as Record<string, unknown>).flatMap(([k, v]) =>
    keyPaths(v, prefix ? `${prefix}.${k}` : k),
  );
}

function keysFor(locale: string): Set<string> {
  const all: string[] = [];
  for (const ns of NAMESPACES) {
    all.push(...keyPaths(i18n.getResourceBundle(locale, ns) ?? {}).map((p) => `${ns}.${p}`));
  }
  return new Set(all);
}

describe("locale key parity", () => {
  const english = keysFor(DEFAULT_LOCALE);
  const others = shippedLocales().map((l) => l.code).filter((c) => c !== DEFAULT_LOCALE);

  it.each(others)("%s has exactly the English key set", (code) => {
    const theirs = keysFor(code);
    const missing = [...english].filter((k) => !theirs.has(k));
    const extra = [...theirs].filter((k) => !english.has(k));
    expect({ code, missing, extra }).toEqual({ code, missing: [], extra: [] });
  });
});
```
