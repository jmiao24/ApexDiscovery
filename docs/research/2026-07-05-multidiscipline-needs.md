# Claude Code 时代各科研大专业的需求 — 需求调研
2026-07-05

> **目的.** 现有 `docs/REQUIREMENTS.md` 的社区证据几乎全部来自生物/基因组学，
> 这本身印证了它自列的最大短板 P1-1（"life-sciences only"）。本文把证据基础
> 从单一学科扩展到全谱系：物理/天文、化学/材料、地学/气候、社会科学，外加
> 生物剩余缺口与**跨学科通用需求**。方法遵循 requirements-analysis：先找真实
> 社区原声（Step 0），再走替换链条 → AI 替代空间 → 护城河 → 评估 → 价值门。
>
> **一句话结论.** 各专业需求 **80% 共享、20% 学科专属**。共享的 80%（可复现/
> 溯源、引用审计、本地算力、HPC、连接器与渲染器框架）正是 `REQUIREMENTS.md`
> 已建好的真正护城河，且天然跨学科。缺的是每个学科的 20% 专属层（格式 I/O、
> 数据连接器、原生查看器、领域纠错闸门），目前只在 bio + climate 上验证过。
> 调研还暴露一个现有文档**未覆盖**的跨学科新需求：**"能跑但科学上是错的"
> 纠错闸门**（见 P0-5 建议）。

## Step 0 — 市场证据（各专业社区原声，verbatim）

**跨学科共识（最强信号）——"能跑但错了"比不能跑更危险：**
- 化学 Venkat Viswanathan："The problem with a large language model is maybe 99%
  is fine, but occasionally, for that 1%, it will do it incorrectly. But, in
  science, you can't have that 1%."（Chemistry World）
- 神经科学 Patrick Mineault：Claude Code "makes it possible to produce wrong
  results faster than ever before"；核心诉求是 "how can I onboard rapidly into
  code generated by the AI, so I can validate it, change it, and reproduce it?"
  （neuroai.science）
- 生物 Tommy Tang（Chatomics）："silent horror bugs, which occur when tests pass
  while the production code is incorrect"；"AI doesn't understand biology — you do."

**物理/天文：**
- 单位/量纲："It loves to say one quantity is 'similar' to another without
  establishing equality."（Tim Andersen）
- 引用："It can't tell me that this formula can be found in this paper"；天文学家
  评测 LLM："if you can find a relevant citation and then you summarize it wrong,
  that's in some ways more insidious"（arXiv 2507.15715）。
- 大数据："plots are embedded in the json file … in base64 format, and that eats
  up large chunks of the context window"（Mineault）；仿真快照（多 GB/TB HDF5）
  根本进不了上下文。

**化学/材料：**
- 幻觉分子：Anthony Gitter："The code that was executed invented a SMILES string
  … When I draw that hallucinated SMILES, it looks nothing like my requested
  Alvespimycin."（X）；blopig："both GPT-4o and o4-mini-high entirely hallucinated
  the structure"。
- 画不出结构：C&EN 咖啡因测试——Grok 画出 "carbons with five bonds"；Gemini 三个
  标签三个不同化学式且都错。
- 幻觉库函数："Claude frequently imagined a function in the RDKit library called
  'rdDecomposition,' which does not exist"（arXiv 2505.18470）。

**地学/气候：**
- 坐标系是头号 bug："treating latitude and longitude as simple floating-point
  numbers without CRS awareness is the source of most geospatial bugs"
  （geospatial-engineer agent）；GeoAnalystBench："a great deficiency in the
  handling of spatial referencing and transformation"。
- 幻觉 API："In the arcpy.mp submodule, ChatGPT will develop code references where
  the submodule components don't even exist."（Esri Community）
- 幻觉数据集：AutoClimDS："failed to autonomously locate authoritative datasets,
  instead hallucinating dataset names or selecting inappropriate sources with
  mismatched temporal/spatial coverage"（arXiv 2509.21553）。
- "能跑但科学上错"：EarthLink："code that runs without error but produces
  scientifically incorrect results due to a subtle misinterpretation"。

**社会科学：**
- 谄媚式误读（头号危险）——Pepinsky："Claude Code seemed to be guessing what I
  wanted to hear, interpreting basic regression results incorrectly but
  provocatively"；其铁律："Use agentic AI for tasks that involve following rules.
  Do not use agentic AI for tasks that generate answers, arguments, or
  interpretations."
- 复现专有软件：ChatGPT 生成 SPSS 语法时混入其它语言命令（Loop、Do Repeat），
  "Zero out of three answers provided usable syntax-code"（Frontiers 2024）；存在
  开源偏向（专有语言文档少）。
- 幻觉引用：心理学各子领域 6–60%；"55% of the GPT-3.5 citations but just 18% of
  the GPT-4 citations are fabricated"。
- 质性研究："ChatGPT has synthesised plausible-sounding quotes from fragments, or
  worse, invented them entirely … LLM chatbots are the wrong tool for qualitative
  data analysis"（skimle）。

**生物剩余缺口：**
- 静默领域 bug："off-by-one … treating 0-based coordinates as 1-based, and strand
  orientation confusion"；BioCoder："even ChatGPT-4 scores only 50% accuracy"；
  "80% of AI-generated bioinformatics code requires human correction"。
- 大文件："memory problems (loading whole genomes instead of streaming)"；人类
  FASTQ ~90 GB、BAM ~160 GB，远超单轮上下文。
- 流水线可复现："If Claude generates an analysis … there's no clear way to audit
  what was done"（bioinformy）。

**现存产品与自建替代方案：**
- 通用 Skills/连接器：`K-Dense-AI/scientific-agent-skills`（140 skills + 100+ DB，
  含 RDKit/pymatgen/OpenMM/Astropy/SymPy）、`GPTomics/bioSkills`、Crawfurd 的经济学
  五件套 Claude 技能、`imbad0202/academic-research-skills`（52 技能含 /lit-review）。
- 学科 MCP：Materials Project 官方 MCP（`mp_api` v0.46）、`SandyYuan/astro_mcp`
  （DESI + 40 astroquery 服务）、多个 NASA ADS/arXiv MCP、Earth Engine MCP、
  GIS Copilot（QGIS 插件，注入工具签名防幻觉）。
- 说明：社区已在各学科**自发造轮子**填 AI 缺口——痛点真实，且方向与本项目一致。

## Layer 1 — 替换链条（按学科）

各学科当前工作流惊人一致：**"人主导 → AI 写脚本驱动成熟领域库 → 人逐步验证输出"**，
明确*不*让模型跑研究本身、也*不*让它下结论。

| 学科 | Who / 现在怎么干 | 最贵/最脆的一步（Pain peak） |
|---|---|---|
| 物理/天文 | 用 Cursor/CC 写分析与仿真脚本；FITS/HDF5/ROOT；LaTeX 重度；ADS 取引用 | 单位/量纲错误、仿真大数据进不了上下文、引用错配 |
| 化学/材料 | LLM 写 Python 驱动 pymatgen/ASE/RDKit；生成 VASP/QE 输入；正则解析 OUTCAR | 从记忆吐 SMILES/POSCAR（幻觉分子/错化学计量）、画不出结构 |
| 地学/气候 | NL→代码做 QGIS/arcpy/GEE/xarray；NetCDF/GRIB/GeoTIFF | CRS/投影错、20GB+ 栅格 OOM、幻觉数据集名 |
| 社会科学 | 上传 CSV/.dta→让 AI 跑回归、清洗、Stata→R；质性上传访谈编码 | 谄媚式误读结果（隐性 p-hacking）、复现 Stata/SPSS、幻觉引用 |
| 生物 | Nextflow/Snakemake 流水线；BAM/CRAM/FASTQ；正/负对照验证 | 坐标 off-by-one/链向、大文件流式、流水线不可复现 |

**Cost（共性数字）**：质性分析 25 次访谈手工编码 40–80 小时；一个曾耗 ~1 个月的
流水线被 agent 自动化；但"80% 生成的生信代码需人工修正"、"AI 辅助多步物理推理
反而降低正确率并增加灾难性失败"。省时是真的，但把关成本也是真的。

## Layer 2 — AI 替代空间（哪些步骤值得替代，提升几倍）

**10x 且值得做（各学科通用）：**
- **格式 I/O + 大文件内省**：不把整文件塞进上下文，而是读 header/schema、采样、
  正则解析大日志返回结构化数字（材料界已验证：VASP OUTCAR 直读会幻觉 20M token
  且失败；用 memory-pointer 仅 1234 token 且成功，16000× 差距）。
- **粘合代码 + 流水线编排**：把成熟库串起来（pymatgen/ASE、xarray/Dask、
  tidyverse/Stata、Snakemake），这是当前最真实的收益点。
- **接数据库**：把"花几个月学 astroquery/earthaccess API"变成"直接说要什么"。

**2x 或有风险，需强约束（别全自动）：**
- 结论/解释/因果判断 → 社科专家明令"do not use agentic AI for interpretations"。
- 引用与数字 → 必须落地到真实 DOI/源，否则 6–60% 造假。
- 领域正确性（单位/CRS/链向/化学计量/统计选型）→ 需确定性闸门，非模型自评。

**AI 需要什么上下文才能做好**：真实库的函数签名（防幻觉）、文件 header/元数据、
学科数据库的"可执行访问路径"（知识图谱式，而非关键词搜索）、领域约束规则。

**替代后留给人**：主导设计、判断科学正确性、下结论与写论断——即"execute, don't
interpret"边界。

## Layer 3 — 护城河

- **飞轮位置**：不在模型，在于 (1) 溯源/可复现记录，(2) 学科连接器 + 技能的
  可插拔生态，(3) 领域纠错闸门积累的规则库。用户越用、连接器/技能/规则越全。
- **切换成本**：工作区 + 溯源历史 + 已配置的连接器/集群/技能构成 workflow 锁定。
- **可防御性**：单个连接器/查看器对手两周能抄；**"跨学科统一的可复现+纠错+
  溯源底座"整套，加上每学科深度，才是护城河**。这恰是本项目已建的方向。
- **与现有资产协同**：项目已有 provenance.jsonl、reviewer 技能、MCP 管理、3D/
  基因组查看器、HPC 卡、图表设计系统——各学科专属层都是在此之上"加插件"，
  不需动核心（符合 P1-1 的架构承诺）。

## Layer 4 — 多角度评估

- **痛点真实度｜高**：五个学科社区都在自发造 Skills/MCP 填 AI 缺口，且抱怨高度
  一致（幻觉、能跑但错、大文件、不可复现）。这是行为证据，不是假设。
- **市场规模｜高**：覆盖全体量化科研人群；经济学家 AI 采用率已 39%，生信/comp
  chem/GIS 社区活跃。
- **差异性｜中→高**：通用底座（可复现+溯源+本地+审计）已领先多数"Jupyter+chatbot"
  套壳；但学科深度目前只在 bio+climate 兑现，物理/化学/社科尚是空白。
- **竞争剧烈｜高**：Claude Science、各家 Skills 库、学科 MCP 井喷。窗口正在关。
- **护城河可建性｜中**：单点易被抄；整套底座+多学科深度难被抄，但需持续投入。
- **时机｜刚好**：agentic 科研 2025–2026 刚成主流，社区工具碎片化、尚无统一
  可复现工作台整合它们——正是整合者的窗口。
- **上手门槛｜中**：本地优先 + 一键连接器降低门槛，但多学科意味着首屏要让非生物
  用户立刻看到"有我的场景"。

## Layer 5 — 价值门结论

本项目是**开源、本地优先**的既有产品，Layer 5 用于给**路线图排序**，而非判断是否
立项。

- **Q1 付费点**：开源本身不直接收费；自然变现在于（未来）团队/机构版、私有部署、
  合规（PHI/未发表数据不出域）。这些付费点依赖"可信+可复现+多学科通用"这一底座。
- **Q2 战略资产**：清晰——(a) 跨学科科研用户社区，(b) 溯源/纠错规则数据飞轮，
  (c) "可复现科学工作台"的品类权威声誉。三者都能转化为下游机构版收入。
- **Q3 下游可盈利**：机构/受限环境（TRE、HPC、药企基因数据）愿为本地+可审计+
  多学科的工作台付费——但前提是它不再"life-sciences only"。

**判定：做（继续），且把重心从"通用底座"转向"多学科深度 + 一个未覆盖的通用纠错
闸门"。** 底座已成，护城河的下一段在广度与领域正确性。

## Entry point — 建议补入 `REQUIREMENTS.md` 的条目

### 新增 P0-5 · 领域正确性闸门（"能跑" ≠ "对"）——现有文档最大缺口

- **证据**：每个学科的头号痛点都是"跑得过但科学上错"：物理单位/量纲、地学 CRS/
  投影、生物 off-by-one/链向、化学化学计量/化合价、社科统计选型 + 谄媚式误读。
  现有 P0-4 reviewer 只查引用/数字/图-码一致性，**不查领域正确性**。
- **需求**：可插拔的领域校验器（每学科一组规则），在执行前后拦截该学科的经典
  错误类；以结构化、可忽略的 finding 呈现，never 保证"无错"。
- **Acceptance**：至少 3 类学科闸门可用并各拦一例——单位不一致（物理）、CRS
  未对齐/lat 当裸浮点（地学）、0/1-based 坐标或链向错（生物）。
- **为何用户会立刻切换**：这直击"99% 没用、就怕那 1%"的信任门槛，是把 agent 从
  "更快地产出错误"变成"可托付"的关键。

### 强化 P1-2 连接器（学科专属数据库，当前空白最大）

- 物理/天文：ADS、SIMBAD、VizieR、MAST/IRSA、Gaia、SDSS/DESI、GWOSC/LIGO。
- 化学/材料：Materials Project（已有官方 MCP，优先接）、PubChem、ChEMBL；缺口
  ICSD、COD、NIST、Reaxys（多为付费/无 API）。
- 地学/气候：NASA Earthdata（earthaccess 认证流）、Copernicus/Sentinel、CDS/ERA5、
  NOAA、USGS、GEE catalog、ESGF/CMIP6——需"可执行访问路径"元数据防幻觉数据集。
- 社科：FRED（已可）、IPUMS API、ICPSR、OSF、GSS、World Bank；文献 OpenAlex/
  Scholar/JSTOR/SSRN 做引用落地。

### 强化 P1-3 渲染器（各学科原生查看器）

- 已有：3D 分子/晶体（3Dmol）、基因组轨道。新增优先级：能带/DOS + 相图（材料）、
  FITS 天图（WCS）/光谱/corner 图（天文）、气候异常图（Cartopy 正确 transform/
  投影/发散色标）、质性编码的**双向溯源视图**（每个 code/theme 链回原文片段、
  绝不造引文）。

### 补 P0-2/P2-1：大文件"引用而非载入"

- 各学科同一失败模式（FITS/HDF5/ROOT、NetCDF/GRIB、BAM/CRAM/FASTQ、VASP 日志、
  .dta）。统一能力：读 header/schema + 采样 + 正则解析大日志返回结构化数字 +
  memory-pointer，绝不把整文件塞进上下文。

### 社科专属：execute-don't-interpret 边界

- 运行分析、surface 原始输出，但对因果/"provocative"解释**默认withhold或显著标注**；
  预注册对照（跑出的流水线 vs 注册的分析计划，标出偏离，防 HARKing）；确定性种子
  + 逐数字溯源（script + 行号 + 输出）。
